# UI Catalog Category Specification

## 1. 문서 목적

이 문서는 UI 카탈로그(`src/catalog/catalog.json`)에서 사용하는 카테고리 사양을 정의한다.
카테고리의 의미를 명확히 하여 검색/필터링 기준을 일관되게 유지하는 것을 목표로 한다.

## 2. 데이터 필드 사양

각 카탈로그 엔트리는 아래 필드를 가진다.

- `id`: 항목 고유 식별자
- `name`: 컴포넌트 표시 이름
- `description`: 컴포넌트 설명
- `tags`: 보조 검색용 태그 배열
- `componentId`: 프리뷰 레지스트리 연결용 ID
- `category`: 주 카테고리(필수)

`category`는 단일 문자열이며, 아래 허용값 중 하나를 사용한다.

## 3. 카테고리 목록 (v1)

- `action`: 버튼/트리거 등 사용자 액션 유도 컴포넌트
- `layout`: 카드/섹션/컨테이너 등 배치 중심 컴포넌트
- `status`: 배지/칩/라벨 등 상태 표현 컴포넌트
- `form`: 입력/선택/토글 등 폼 관련 컴포넌트
- `navigation`: 탭/메뉴/브레드크럼 등 이동 구조 컴포넌트
- `feedback`: 토스트/배너/로딩 등 피드백 컴포넌트
- `overlay`: 모달/드로어/툴팁 등 오버레이 컴포넌트
- `data-display`: 테이블/리스트/통계 등 데이터 표현 컴포넌트

## 4. 네이밍 규칙

- 카테고리 값은 영문 소문자 `kebab-case`를 사용한다.
- 공백, 대문자, 한글 키워드는 카테고리 값으로 사용하지 않는다.
- UI 노출 시에는 별도의 라벨 매핑으로 한글명을 표시한다.

예시 라벨 매핑:

- `action` -> `액션`
- `layout` -> `레이아웃`
- `status` -> `상태`

## 5. 기존 항목 매핑 예시

- `Accent Button` -> `action`
- `Promo Card` -> `layout`
- `Badge Stack` -> `status`

## 6. 필터링 동작 기준

- 기본값은 `all`이며 전체 항목을 표시한다.
- 특정 카테고리 선택 시 `entry.category`가 일치하는 항목만 표시한다.
- `tags`는 카테고리를 대체하지 않으며, 보조 검색/분류 용도로만 사용한다.

## 7. 확장 가이드

- 신규 카테고리 추가 시, 이 문서의 목록과 라벨 매핑을 함께 업데이트한다.
- 카테고리 수가 늘어날 경우 우선 기존 카테고리와의 중복 의미를 검토한다.
- 다중 카테고리가 필요한 경우, `category`를 유지한 채 추후 `categories: string[]` 확장을 검토한다.
